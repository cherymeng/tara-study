**# iPad / 平板端 UI 设计规范**

本项目最终主要运行在 iPad / 平板设备上，因此 MVP 阶段采用：

\`\`\`text
平板优先
横屏优先
触控优先
儿童友好
大卡片布局
低层级导航
即时反馈
\`\`\`

---

# 1. 目标设备

优先适配：

\`\`\`text
iPad 10.2 / 10.9 / 11 inch
iPad Air
iPad Pro 11 inch
Android Tablet
\`\`\`

推荐设计基准：

\`\`\`text
宽度：1024px \~ 1366px
高度：768px \~ 1024px
方向：横屏优先
\`\`\`

---

# 2. 页面整体结构

平板端不采用手机式底部导航，而采用三栏结构：

\`\`\`text
┌──────────────────────────────────────────────┐
│ 左侧导航 │        主内容区域        │ 成长反馈区 │
│          │                         │           │
│ 首页     │ 今日任务 / 世界 / 宠物    │ 宠物状态   │
│ 任务     │                         │ XP / 等级  │
│ 宠物     │                         │ 连续打卡   │
│ 世界     │                         │ 成就提示   │
│ 家长     │                         │           │
└──────────────────────────────────────────────┘
\`\`\`

---

# 3. 布局尺寸

## 页面外层

\`\`\`text
最小宽度：1024px
推荐最大宽度：1366px
页面 padding：24px
模块间距：20px \~ 32px
\`\`\`

## 左侧导航

\`\`\`text
宽度：88px \~ 120px
按钮尺寸：64px x 64px
图标优先
文字辅助
\`\`\`

## 主内容区

\`\`\`text
自适应宽度
用于承载主要任务、地图、宠物、报告
\`\`\`

## 右侧成长反馈区

\`\`\`text
宽度：260px \~ 320px
用于显示：

- 宠物
- 等级
- XP
- 连续打卡
- 今日鼓励
  \`\`\`

---

# 4. 触控规范

所有关键操作都要适合儿童手指点击。

\`\`\`text
主要按钮高度：56px \~ 72px
任务卡片高度：120px \~ 180px
最小点击区域：48px x 48px
推荐点击区域：64px x 64px
卡片圆角：24px \~ 32px
\`\`\`

避免：

\`\`\`text
过小文字
密集表格
复杂表单
多级菜单
长文本输入
\`\`\`

---

# 5. 儿童友好原则

## 操作要简单

\`\`\`text
点击完成任务
点击领取奖励
点击查看宠物
点击解锁世界
\`\`\`

## 一屏完成

核心任务尽量在一屏内完成，避免频繁跳转。

## 即时反馈

完成任务后立即展示：

\`\`\`text
XP +20
宠物开心动画
等级进度条增长
成就提示
\`\`\`

---

# 6. 页面优先级

MVP 首页应优先显示：

\`\`\`text
今日任务
宠物状态
成长进度
世界解锁进度
\`\`\`

不应优先显示：

\`\`\`text
复杂统计
长篇学习报告
设置项
AI功能入口
\`\`\`

---

# 7. 推荐页面结构

## 首页

\`\`\`text
左侧：导航
中间：今日任务 + 成长目标
右侧：宠物 + 等级 + 连续打卡
\`\`\`

## 任务页

\`\`\`text
中间：任务卡片网格
右侧：完成奖励预览
\`\`\`

## 宠物页

\`\`\`text
中间：宠物大展示
右侧：宠物等级 / 心情 / 装饰
\`\`\`

## 世界页

\`\`\`text
中间：世界地图
右侧：解锁条件 / 当前进度
\`\`\`

## 家长页

\`\`\`text
中间：成长报告
右侧：本周摘要
\`\`\`

---

# 8. Tailwind 响应式建议

项目以平板为主，不强制做复杂移动端。

推荐断点：

\`\`\`text
tablet: 768px
desktop: 1024px
wide: 1280px
\`\`\`

推荐布局：

\`\`\`text
grid-cols-[96px\_1fr\_300px]
\`\`\`

示例：

\`\`\`tsx

<div className="min-h-screen grid grid-cols-[96px\_1fr\_300px] gap-6 p-6">
  <SidebarNav />
  <main>
    <Outlet />
  </main>
  <GrowthPanel />
</div>
\`\`\`

---

# 9. 视觉风格

推荐：

\`\`\`text
明亮
柔和
圆角
大面积留白
卡片化
轻动画
高对比度按钮
\`\`\`

避免：

\`\`\`text
成人后台风格
灰色表格风格
过多文字说明
过度复杂数据图表
\`\`\`

---

# 10. MVP 验收标准

平板端 MVP 应满足：

- [ ]  横屏下主要功能一屏可见
- [ ]  儿童可以独立完成任务打卡
- [ ]  主要按钮足够大
- [ ]  成长反馈即时可见
- [ ]  宠物始终有存在感
- [ ]  家长入口不干扰儿童主流程
- [ ]  不依赖复杂键盘输入
- [ ]  iPad 浏览器中可正常使用
