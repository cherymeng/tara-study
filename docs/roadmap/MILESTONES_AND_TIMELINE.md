# 桐宝学习 - 项目里程碑与开发时间线（Milestones & Timeline）

# 1. 文档目标

本文件用于：

* 明确产品开发阶段
* 拆解里程碑
* 统一开发节奏
* 指导团队执行
* 支持长期迭代规划

---

# 2. 产品开发核心原则

桐宝学习 的开发原则：

# 小步快跑

# 持续成长

# 逐步进化

不是：

* 一次性做完整产品

而是：

# 先验证核心成长循环。

---

# 3. 整体阶段规划

项目整体分为：

| 阶段      | 名称     | 目标       |
| ------- | ------ | -------- |
| Phase 1 | MVP阶段  | 验证成长循环   |
| Phase 2 | 成长系统阶段 | 提高长期留存   |
| Phase 3 | 探索成长阶段 | 建立差异化    |
| Phase 4 | AI能力阶段 | 建立AI成长伙伴 |
| Phase 5 | 长期生态阶段 | 建立成长平台   |

---

# 4. Phase 1 - MVP阶段

# 目标：

验证：

> 孩子是否会因为成长感而持续学习。

---

## 核心功能

包括：

* 用户系统
* 今日任务
* 积分系统
* 等级系统
* 连续成长
* 基础宠物
* 家长报告
* 基础成就

---

## 页面

包括：

### 儿童端

* 首页
* 今日任务页
* 成长页
* 宠物页

---

### 家长端

* 家长首页
* 基础学习报告

---

## 技术目标

包括：

* React/Vite工程
* 基础状态管理
* LocalStorage
* 基础动画系统

---

## 成功标准

包括：

* 孩子愿意连续使用
* 家长认可成长价值
* 积分反馈有效
* 成长感成立

# 12. 项目管理原则

开发过程中：

必须：

# 始终围绕核心成长循环。

避免：

* 功能堆积
* 复杂化
* 过早AI化

让孩子：

* 愿意学习
* 愿意探索
* 愿意成长
